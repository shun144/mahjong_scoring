# OBJECTIVE.md — テンポ訴求の目的と成果物

本ドキュメントは、本アプリの差別化（**テンポの良さ**）をどう成立させ、どう訴求するかの
**方針と成果物**をセッション跨ぎで保持するための正典。実装計画の詳細タスクは `docs/TASKS.md`、
仕様は `docs/SPEC.md`、デザインは `docs/DESIGN.md` を参照。

---

## 1. 差別化の核と位置づけ

- **差別化＝テンポの良さ**。競合は「1問ごとにスクロールして次へ進む／同じ問題が繰り返される」。
  本アプリは**サクサク回せる**ことを売りにする。
- **二段構えのポジショニング**:
  - テンポ＝**フック**（入口の吸引力）。
  - 正確さ・信頼（正確な点数エンジン／全役・役満網羅／符計算対応／苦手を自動復習）＝**クローザー**
    （定着＝再訪＝広告収益の本体）。
  - テンポ単独では戦わない。「テンポで惹き、正確さで刺す」。
- **テンポは集客の武器ではなく、着地後の定着×マネタイズの武器**。
  - 体感品質なので検索スニペットでは伝わらず、検索意図とも噛み合わない。
  - 広告収益モデルでは 1セッションの問題数＝PV＝広告表示 なので、テンポ（周回のしやすさ）が収益に直結。

## 2. テンポの定義（何を「サクサク」と呼ぶか）

- テンポ＝**「答えに速く着く＋理解に速く着く」**。解説は寄り道ではなく*テンポの一部*として内側に抱える。
- 成立条件（＝Bの受け入れ条件。詳細は `docs/TASKS.md` T-006〜T-008）:
  1. **遷移ゼロ** — 回答結果を出題画面にインライン表示し、`/quiz→/result→/quiz` の2遷移を廃止。
  2. **解説は条件付き** — 正解時は畳む（テンポ優先）、不正解時は自動展開（学ぶべき瞬間に内訳）。非懲罰。
  3. **無スクロール1画面** — 正解時の結果が1画面に収まる。
  4. **ワンタップ次へ** — ノーロードで次問。
  5. **モメンタム可視化** — 「今日の回答数」を主役・大きく、「連続正解数」を非懲罰の脇役・小さく。
     不正解でも回答数は減らさない。

## 4. ② C：ループ動画 絵コンテ ＋ OGP案

テンポは*動き*でしか伝わらず、リンクプレビューは*静止画*しか出ない。この2面を埋める。
現状 `index.html`/`RouteMeta` に *_og:_ タグは皆無**なので、OGP整備はゼロから。

### 4.1 ループ動画（GIF/短尺）の絵コンテ

「タップ→その場で答え＋カウンタ加算→次」の**ノーロード1周**を映す。無スクロール・無遷移を強調。

1. 出題画面（手牌・アガリ牌・ドラ・「今日の回答数」カウンタが見えている）。
2. 4択を1つタップ → **その場で**○正解＋答えが出る（画面遷移なし）。
3. カウンタ「今日の回答数」が **+1** に弾む（連続正解も小さく+1）。
4. 「次へ」を1タップ → **即・次問**（ロード無し）。
5. これを数問テンポよく連続（正解ラン）。1回だけ不正解→解説が自動展開→「なるほど」→次、を混ぜると誠実。

- **原則**: 「テンポは語るな、味わわせて、その様子を切り取って見せろ」。文字テロップは最小限。
- **用途**: LPヒーロー、SNS投稿。**T-006 の画面録画がそのまま素材**（＝B検証の証拠映像と兼用）。

### 4.2 OGP静止画の構図案

- 1問の綺麗な出題画面＋「今日の回答数」カウンタ（大きな数字）を主役に。
- リンクプレビュー（X/LINE/Discord）で「サクサク解けるドリル」と一目で分かる構図。
- 整備物: `og:title` / `og:description` / `og:image` / `twitter:card`（現状全て未設定）。

## 5. ③ 集客コピー方針

- **集客コピーにテンポを載せない**。検索意図（無料・反復・符/翻・正確）で最適化する現状方針を維持。
  - 現行 `/` の title 例: 「…点数計算(符・翻)を反復練習できる無料ドリル」／
    description「…点数計算(符・翻・点数)をスマホで反復練習できるドリルアプリ。」→ **この路線を維持**。
  - テンポは着地後の体験と §4 のループ動画で証明する。
- **ホームに信頼ストリップを軽く添える**（テンポは大きく謳わない）:
  - 文言案: 「正確な点数エンジン／全役・役満まで網羅／符計算にも対応／苦手を自動で復習」。
  - あくまで*補助*。ホームは低レバレッジ（SEO流入の多くは /quiz・記事に着地）なので過投資しない。
- **信頼はコピーではなく解説の質で語らせる**（T-007 で不正解時に展開される内訳の作り込み）。

## 6. 順序と前提

- **順序**: 構造（T-006 インライン化）を先に着手済み方針。嗜好依存層（解説デフォルト T-007・カウンタ T-008）は
  現ビルドへの体感レビュー結果で微調整する余地あり（※ユーザー判断で「全部今作る」を選択済み。逆の声が来たら
  嗜好層のみ載せ替え）。
- **体感レビューの位置づけ**: 現ビルド（従来フロー：別ルート/result・解説常時表示・カウンタ無し）への実ユーザ評価が
  これから返る。§2 の変更群は現ビルド未搭載の「次バージョン仮説」。
