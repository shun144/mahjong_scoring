# 麻雀 点数計算 練習アプリ — 仕様書（SPEC）

> 本書はアプリの唯一の正典（source of truth）。実装・タスクは本書に従う。ドキュメントは日本語で維持する。

---

## 1. 目的・対象ユーザー・ゴール

- **目的**: 麻雀の点数計算を、スマホWebで反復練習できるドリルアプリを提供する。
- **対象ユーザー**: 点数計算を覚えたい初〜中級者。実戦で「何点か即答できる」状態を目指す人。
- **ゴール**:
  - ランダム出題された和了形を見て、正しい**最終点数**を素早く選べるようになる。
  - 間違えたときに**なぜその点数か**（役・翻・符・計算式）を理解できる。
  - **苦手分野**（符の型・役の種類）を可視化し、重点的に復習できる。

---

## 2. スコープ

### 2.1 含む
- 四人麻雀・標準リーチ麻雀の和了点計算
- 門前手／鳴き手（副露・喰い下がり）
- 七対子・特殊符（平和20符・七対子25符 等）
- 役満・数え役満
- 赤ドラ・ドラ・裏ドラ
- リーチ有無、門前ツモ、自風・場風・親子の条件
- 高点法（複数解釈時は最高点を正解）

### 2.2 含まない（スコープ外）
- **状況役**: 一発・海底摸月・河底撈魚・嶺上開花・槍槓・ダブルリーチ・天和・地和・人和・流し満貫
- **ハウス/ローカルルール**: 切り上げ満貫・ダブル役満・役満の複合・青天井
- 三人麻雀（サンマ）、両対応
- 持ち点・ウマ・オカ・順位・供託（リーチ棒/本場）による加算
- タイマー／タイムアタック
- PWA／オフライン対応
- 多言語対応（日本語のみ）

> 注: 状況役は手牌から推測不能なため除外。門前ツモとリーチは条件として与えるため含む。

## 3. 用語集

| 用語 | 意味 |
|---|---|
| 翻（ハン） | 役やドラの合計値。点数の指数的な係数。 |
| 符（フ） | 手の形から算出する加算値。10符単位に切り上げる。 |
| 親／子 | 親=場に対する起家（自風が東の家）。ロン・ツモの支払いと点数が異なる。 |
| ツモ／ロン | 自摸和了／他家からの栄和。符と支払い形式が変わる。 |
| 面前（門前） | 鳴いていない状態。門前でのみ成立する役がある（リーチ・平和・七対子等）。 |
| 鳴き（副露） | ポン・チー・明槓で他家の牌を晒すこと。喰い下がりが発生する。 |
| 喰い下がり | 鳴くと翻が下がる役（三色2→1 等）。 |
| ドラ／裏ドラ／赤ドラ | 翻を増やすボーナス牌。**役ではない**（ドラのみでは和了不可）。 |
| ドラ表示牌／裏ドラ表示牌 | 実際のドラ／裏ドラを示す牌（次の牌がドラになる）。プレイヤーは表示牌からドラを読み替える必要がある。 |
| 高点法 | 1つの手が複数解釈できるとき、**最も高い点数**になる解釈を正とするルール。 |
| 数え役満 | 13翻以上を役満（子32000/親48000）として扱う。 |

## 4. 機能要件

### 4.1 出題
- 問題は**ランダム**に提示する（バンクからのシャッフル＋ジェネレータ生成）。
- 出題画面に表示する情報:
  - **手牌**（純手牌＋副露があれば副露を横向きで表示）
  - **上がり牌**（ツモ牌／ロン牌）
  - **ツモ／ロンの別**
  - **ドラ表示牌**（実戦準拠。プレイヤーが表示牌から実際のドラを読み替える）
  - **裏ドラ表示牌**（リーチ時のみ提示。非リーチ時は非表示）
  - **自風**（東南西北）、**場風**（東 or 南）
  - **親／子**
  - **リーチ有無**
- **局条件の範囲**: 場風は東・南のみ。自風は東南西北。西場・北場は出題しない。
- すべての出題は**最低1役以上が成立**する正当な和了形であること（ドラのみの手は出さない）。

### 4.2 回答UI
- 回答は**最終点数のみ**を選択肢ボタンから選ぶ。
- **選択肢の形式は親/子・ツモ/ロンで変わる**:
  - ロン: 単一値（例: `8000`）
  - 子ツモ: `子X / 親Y`（例: `2000 / 4000`）
  - 親ツモ: `X オール`（例: `4000 オール`）
- 選択肢は**正解＋紛らわしい誤答の4択**（§7の方針で生成）を4つのボタンとして並べる。
- ボタンを押すと即座に採点し、正誤にかかわらず解説画面へ遷移する（別途の「回答する」ボタンは持たない）。

### 4.3 採点
- **高点法**で正解を決定（複数解釈のうち最高点）。
- 選択された点数が正解の支払い（形式含む）と一致すれば正答。

### 4.4 解説
- 正誤（○×）と正しい点数。
- **内訳**: 成立役の一覧（役名＋翻）、合計翻、符、計算式（例: `40符3翻 → 子ロン 5200`）。
- **符の内訳**: 符計算の各要素（副底・門前ロン加符・ツモ符・面子の符・雀頭の符・待ちの符）を1行ずつ列挙し、切り上げ前の小計と切り上げ後の最終符を示す。待ちの形は +0（両面・双碰）でも明示する。平和・七対子・喰い平和形などの固定・特殊符はその旨を注記する。満貫以上（符が点数に影響しない区分）では符の内訳は表示しない。
- **高点法の別解**: 他の解釈が存在する場合、別解とその点差を提示（「なぜ高い方を採るか」）。

### 4.5 成績・苦手復習
- **成績**を localStorage に保存: 累計正答率・履歴・現在の連続正解・最高連続正解。
- **苦手復習**: ミスを **符の型** と **役の種類** の2軸で記録し、正答率の低いタグの問題を重み付けして出題する。
- 成績画面で正答率・連続正解・苦手分野を可視化する。

## 5. 点数計算ルール仕様

> エンジンはこの節を厳密に実装する。教材のため**誤答提示は致命的**。ユニットテストで担保する。

### 5.1 対象役リスト（翻数・喰い下がり）

**1翻**
- リーチ（門前のみ）
- 門前清自摸和（ツモ、門前のみ）
- 平和（門前のみ）
- 断幺九（タンヤオ）
- 一盃口（門前のみ）
- 役牌: 白・發・中／自風牌／場風牌（各1翻。連風牌は§5.1補足）

**2翻**（かっこ内は喰い下がり後）
- 三色同順（1）
- 一気通貫（1）
- 混全帯幺九／チャンタ（1）
- 七対子（門前のみ、25符固定）
- 対々和
- 三暗刻
- 三色同刻
- 三槓子
- 混老頭
- 小三元

**3翻**
- 混一色（喰い2）
- 純全帯幺九／純チャン（喰い2）
- 二盃口（門前のみ）

**6翻**
- 清一色（喰い5）

**役満**（すべて1倍。複合・ダブル役満は不採用）
- 国士無双／四暗刻／大三元／字一色／緑一色／清老頭／小四喜・大四喜／四槓子／九蓮宝燈

**補足**
- **連風牌**（自風＝場風の刻子。例: 東場の東家の東刻）: 標準ルールに従い雀頭2符／刻子は通常の刻子符。役としてはダブル役牌＝2翻（場風1＋自風1）として扱う。
- **数え役満**: 翻の合計が13翻以上で役満扱い（子32000／親48000）。
- ドラ・裏ドラ・赤ドラは翻に加算するが役ではない（単独では和了不可）。

### 5.2 符計算

**加符要素**
| 要素 | 符 |
|---|---|
| 副底（基本） | 20 |
| 門前ロン加符（門前かつロン） | +10 |
| ツモ（平和ツモを除く） | +2 |
| 待ち: 単騎／嵌張／辺張 | +2 |
| 待ち: 両面／双碰（シャンポン） | +0 |
| 雀頭: 三元牌／自風牌／場風牌 | +2 |

**面子の符**
| 面子 | 中張牌(2-8) | 幺九牌(1,9,字) |
|---|---|---|
| 順子 | 0 | 0 |
| 明刻 | 2 | 4 |
| 暗刻 | 4 | 8 |
| 明槓 | 8 | 16 |
| 暗槓 | 16 | 32 |

**特殊符（固定・優先）**
- **平和ツモ**: 20符固定（ツモ符も待ち符も付かない門前平和形）。
- **平和ロン**: 30符（副底20＋門前ロン10）。
- **七対子**: 25符固定。
- **喰い平和形のロン**（鳴きで符が一切付かない20符形）: 20符では成立しないため**30符に切り上げ**る。

**丸め**
- 上記を合算し、**10符単位に切り上げ**（一の位を切り上げ）。

### 5.3 点数計算

**基本点** = 符 × 2^(2 + 翻)

支払い（各支払いは**100点単位に切り上げ**）:
| | 子（非親） | 親 |
|---|---|---|
| ロン | 基本点 × 4 | 基本点 × 6 |
| ツモ | 子から 基本点×1、親から 基本点×2 | 各家から 基本点×2 |

**満貫以上（基本点に上限を設定）**
| 区分 | 適用翻 | 子（ロン / ツモ） | 親（ロン / ツモ） |
|---|---|---|---|
| 満貫 | 4翻40符以上・3翻70符以上・5翻 | 8000 / 2000・4000 | 12000 / 4000オール |
| 跳満 | 6〜7翻 | 12000 / 3000・6000 | 18000 / 6000オール |
| 倍満 | 8〜10翻 | 16000 / 4000・8000 | 24000 / 8000オール |
| 三倍満 | 11〜12翻 | 24000 / 6000・12000 | 36000 / 12000オール |
| 役満 | 役満役・数え役満(13翻+) | 32000 / 8000・16000 | 48000 / 16000オール |

**代表的な点数（満貫未満、参考）**
- 子ロン: 30符1翻1000／2翻2000／3翻3900／4翻7700、40符2翻2600／3翻5200、25符2翻1600／3翻3200／4翻6400
- 親ロン: 30符1翻1500／2翻2900／3翻5800、40符3翻7700
- （その他はすべて基本点の式から算出）

### 5.4 ドラの数え方
- **ドラ表示牌→ドラの変換ルール**（実戦準拠）:
  - 数牌（萬子/筒子/索子）: 表示牌の次の数字がドラ（例: 表示牌4索 → ドラは5索）。9の次は1に戻る（例: 表示牌9萬 → ドラは1萬）。
  - 風牌（字牌）: 東→南→西→北→東 の周期でドラが決まる（例: 表示牌東 → ドラは南）。
  - 三元牌（字牌）: 白→發→中→白 の周期でドラが決まる（例: 表示牌白 → ドラは發）。
- **ドラ**: 上記変換で得た実際のドラ牌が手牌（副露含む）に含まれる枚数だけ翻を加算。
- **裏ドラ**: リーチ時のみ。裏ドラ表示牌を同じ変換ルールで読み替え、手牌に含まれる枚数だけ加算。
- **赤ドラ**: 手牌中の赤5（赤牌）の枚数だけ加算。各色5に1枚（計3枚）を想定。ドラ表示牌の読み替えとは無関係。

### 5.5 高点法
- 手牌が複数の面子構成・待ち解釈を持つ場合、成立役・翻・符をそれぞれ算出し、**最終点数が最大**になる解釈を正解とする。
- 解説では別解と点差を提示する。

## 6. 問題データモデル

TypeScript のインターフェース（`src/engine` / `src/data` で共有）:

```ts
type Suit = "m" | "p" | "s" | "z"; // 萬子/筒子/索子/字牌
// z: 1=東 2=南 3=西 4=北 5=白 6=發 7=中
interface Tile { suit: Suit; rank: number; red?: boolean; }

type MeldType = "chi" | "pon" | "minkan" | "ankan";
interface Meld { type: MeldType; tiles: Tile[]; }

type Wind = "east" | "south" | "west" | "north";
type WinType = "tsumo" | "ron";

interface Problem {
  id: string;
  source: "bank" | "generated";
  hand: {
    concealed: Tile[];   // 純手牌（副露を除く。上がり牌を含む完成形13+1）
    melds: Meld[];       // 副露
    winningTile: Tile;   // 上がり牌
    winType: WinType;
  };
  doraIndicators: Tile[];    // ドラ表示牌（実際のドラは§5.4の変換ルールで導出）
  uraDoraIndicators: Tile[]; // 裏ドラ表示牌（リーチ時のみ、非リーチは空配列）
  conditions: {
    seatWind: Wind;      // 自風
    roundWind: Wind;     // 場風（east | south のみ）
    isDealer: boolean;   // 親=true
    riichi: boolean;
  };
  answer: ScoreResult;   // §5に基づく正解（高点法適用後）
  tags: {
    fuType: number;              // 20/25/30/40/50/60/70...
    yakuCategories: string[];    // 苦手復習用の役カテゴリ
  };
}

interface YakuResult { name: string; han: number; }

type Payment =
  | { kind: "ron"; total: number }                       // ロン: 単一値
  | { kind: "tsumo-ko"; nonDealer: number; dealer: number } // 子ツモ: 子X/親Y
  | { kind: "tsumo-oya"; each: number };                 // 親ツモ: Xオール

interface ScoreResult {
  yaku: YakuResult[];
  han: number;
  fu: number;
  payment: Payment;
  rank?: "mangan" | "haneman" | "baiman" | "sanbaiman" | "yakuman";
  interpretationNote?: string; // 高点法の別解説明
}
```

- 問題バンクは上記 `Problem` の配列を JSON で持つ。
- `tags` は苦手復習の集計キー。`fuType` は §5.2 の丸め後の符、`yakuCategories` は成立役の分類（例: `"平和"`, `"対々和"`, `"混一色"`, `"役満"` など）。

## 7. 誤答（distractor）生成方針

正解 `(han, fu, payment)` から、**ありがちな誤り**を再現した誤答を生成する:
- 符を**切り上げずに**計算した点数
- **翻±1**（役の数え間違い）
- **親／子を取り違え**た点数
- **ツモ／ロンを取り違え**た点数（同じ手の別支払い形式）
- **ドラ枚数±1**
- 一つ上／下の区分（満貫↔跳満 等）

生成した候補は、正解と同じ**支払い形式（親/子・ツモ/ロン）に揃え**、重複を排除し、正解を必ず含めて**4個**にした上でシャッフルする。値は実在し得る点数に丸める。

## 8. 画面仕様

### 8.1 画面一覧
1. **ホーム**: 練習開始・成績への導線。
2. **出題画面**: §4.1 の情報表示＋点数プルダウン＋回答ボタン。
3. **解説画面**: ○×・正解点数・内訳（役/翻/符/式）・高点法の別解・次の問題へ。
4. **成績画面**: 累計正答率・連続正解・苦手分野（符の型／役の種類別の正答率）。

### 8.2 レスポンシブ方針
- **スマホ最優先**（縦画面）。手牌は折り返しなく1〜2段で視認できるサイズ。
- PCでは中央寄せのカード幅で表示（最大幅を設定）。
- タップターゲットは十分な大きさ（プルダウン・ボタン）。

### 8.3 デザイン
- **モダン・ニュートラル**: 白基調のクリーンなカード型UI。牌画像が主役で可読性重視。
- 牌は**画像スプライト**。鳴き牌は横向き、赤ドラは赤色で表現。

---

## 9. 非機能要件
- **対応ブラウザ**: モバイル/デスクトップの最新版 Chrome・Safari・Edge・Firefox。
- **パフォーマンス**: 出題→表示は体感即時（問題生成は事前/非同期でも可）。エンジンの1問採点は数ms〜十数ms目標。
- **正確性**: エンジンは §5 をユニットテストで網羅（問題バンクを正解データに使う回帰テスト）。
- **アクセシビリティ**: 牌画像に代替テキスト、色のみに依存しない情報提示、フォーカス可能なUI。
- **配信**: 静的ホスティング（GitHub Pages 等）。

---

## 10. 受け入れ基準（Acceptance）
- ランダム出題→回答→採点→解説→成績、の一連が動作する。
- 選択肢が親/子・ツモ/ロンで正しい形式に切り替わる。
- 高点法で正解が決定され、解説に内訳と別解が表示される。
- 苦手分野が符の型・役の種類の2軸で記録・可視化される。
- §2.2 のスコープ外要素が混入していない（状況役・ハウスルール等）。
- エンジンのユニットテストが全て通り、問題バンクの正解と一致する。
