TASKS.md から No.$ARGUMENTS のタスクを読み、実装する。

## 手順

1. 該当タスクの「目的」「確定した設計判断」「影響ファイル」「受け入れ基準」を確認
2. 前提タスク(「〜の土台」等の依存記述)が未完了なら報告して停止
3. 「確定した設計判断」に書かれた方針に**忠実に**実装する(独自解釈で変えない。曖昧な点は都度確認)
4. 「影響ファイル」に挙がっていないファイルまで変更が及ぶ場合は理由を報告
5. 実装完了後、「受け入れ基準」を1項目ずつ検証
   - 各項目について満たしたかどうかを○/✕で報告
   - コマンドで機械的に検証できるもの(`npm test`, `npm run lint`等)は実行して結果を貼る
   - 目視/手動確認が必要なもの(画面録画可否など)は検証不可な旨を明記し、確認手順を提示する
6. 全基準を満たしたらTASKS.mdの該当項目にステータス(例: `- [x]` や「完了」)を追記
7. 満たさない基準があれば、実装済み範囲を明示した上で残タスクを報告(未完了のままステータス更新しない)
